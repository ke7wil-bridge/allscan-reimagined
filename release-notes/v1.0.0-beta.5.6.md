# AllScan Reimagined v1.0.0 Beta 5.6

Beta 5.6 prevents a companion TGIF connected-client collector from accumulating abandoned Socket.IO clients after reconnect failures.

## Changes

- Detects and repairs the known reconnect loop in `/usr/local/sbin/connected-clients-daemon.py` without replacing unrelated or customized daemon code.
- Explicitly disconnects and shuts down a failed Socket.IO client before retrying.
- Prevents a disconnected session loop from continuing to emit requests.
- Adds conservative systemd memory protection for the companion collector: 128 MB high-pressure threshold and 192 MB hard limit.
- Preserves the original daemon once as `connected-clients-daemon.py.pre-asr-beta5.6` before applying the repair.
- Rechecks the repair during ASR's five-minute integrity pass so a later external update cannot silently restore the vulnerable loop.
- Updates the ASR main page and administration pages to display `v1.0.0 Beta 5.6`.

## Verified failure signature

The KE7WIL collector had grown from a normal baseline of approximately 33 MB and 4 threads to approximately 780 MB and 76 threads after 41 TGIF session errors. Restarting only the collector returned it to the normal baseline without restarting Asterisk.
