# AllScan Reimagined v1.0.0 Beta 5.8

Beta 5.8 fixes Favorite names added through ASR.

## Changes

- Looks up the current node label in the local AllStar database before writing a new Favorite.
- Falls back to the node number when no database record is available.
- Dynamically enriches existing exact number-only placeholders such as `29332 29332` when the local database later contains a name.
- Keeps the database name, description, and location in their correct Favorites table columns.
- Preserves manually customized Favorite labels and does not rewrite the Favorites file merely to refresh the display.
- Retains the Beta 5.7 companion-collector maintenance timer and all earlier preservation and topology safeguards.
- Updates the ASR main and administration pages to display `v1.0.0 Beta 5.8`.

## Verified defect

On KE7WIL, ASR displayed node 29332 as `29332`. Original AllScan on Del Webb displayed the same newly added Favorite as `WL7LP`, `Alaska AllStar Hub`, `North Pole, Alaska`. The difference was traced to ASR writing a node-number placeholder instead of consulting its local node database.

```bash
set -e

expected="65f463870eb586b71a2c51812027ae6da2d2d926f363ca3a1e1645435451fbb8"
stage="/tmp/asr-beta-5.8-install"

curl -fL "$url" -o "$pkg"
printf '%s  %s\n' "$expected" "$pkg" | sha256sum -c -

mkdir -p "$stage"

cd "$stage/allscan-reimagined-1.0.0-beta.5.8"

python3 payload/scripts/asr-patch-connected-clients.py --self-test
bash ./install.sh
```
