# AllScan Reimagined v1.0.0 Beta 6

Beta 6 is the first side-by-side AllScan Reimagined release. It keeps the
official stock interface at `/allscan/` and installs ASR separately at `/asr/`.
It remains a prerelease.

## Install or upgrade

Run this from an interactive root terminal. The installer preserves the
existing users, passwords, permissions, Favorites, database, node settings,
Reimagined settings, bridge settings, friendly names, map cache, uploaded
logos, and protected secrets.

```bash
set -e

base="https://github.com/ke7wil-bridge/allscan-reimagined/releases/download/v1.0.0-beta.6"
pkg="/tmp/allscan-reimagined-1.0.0-beta.6.tar.gz"
expected="e2660587da08fca39380c6ce2c67402d27c0cf43b7384ff12dc667916cf66972"
stage="/tmp/asr-beta-6-install"

curl -fL "$base/$(basename "$pkg")" -o "$pkg"
printf '%s  %s\n' "$expected" "$pkg" | sha256sum -c -

rm -rf "$stage"
mkdir -p "$stage"
tar -xzf "$pkg" -C "$stage"
cd "$stage/allscan-reimagined-1.0.0-beta.6"

php -l payload/server/asr-api.php
php -l payload/compat/allscan-v1.01/asr-settings/index.php
php -l payload/compat/allscan-v1.01/asr-instructions/index.php
php -l payload/scripts/asr-bridge-clients.php
php payload/scripts/asr-bridge-clients.php --self-test
php payload/scripts/asr-runtime-source-self-test.php
php payload/scripts/asr-lookup-map-self-test.php
php payload/scripts/asr-access-policy-self-test.php

sh -n payload/scripts/asr-asterisk-read.sh
bash -n payload/scripts/asr-reapply.sh
bash -n payload/scripts/asr-integrity-check.sh
bash payload/scripts/asr-favorites-permissions.sh --self-test
bash payload/scripts/asr-side-by-side-self-test.sh

python3 payload/scripts/asr-patch-connected-clients.py --self-test
python3 payload/scripts/asr-patch-allscan-index.py --self-test
python3 payload/scripts/asr-migrate-tgif-environment.py --self-test
python3 payload/scripts/asr-rollback.py self-test
python3 payload/scripts/asr-installer-rollback-self-test.py --self-test
python3 payload/scripts/asr-bridge-control.py --self-test
python3 payload/scripts/asr-favorites-update.py --self-test
python3 payload/scripts/asr-favorites-source.py --self-test
python3 payload/scripts/asr-instructions-self-test.py
python3 payload/scripts/asr-stock-count-helper.py --self-test

bash ./install.sh
```

Beta 6 installs the original stock interface at `/allscan/` and AllScan
Reimagined at `/asr/`. The installer explains both paths and can require login
for stock `/allscan/` monitoring when that option is selected.

## Migration compatibility

- Omits obsolete `astdb.txt.*` symbolic-link aliases from legacy overlay
  rollback archives while preserving the real Asterisk database and canonical
  Favorites source.
- Keeps strict archive validation for unknown links, hard links, special files,
  unsafe paths, oversized files, and files outside the expected webroot.
- Uses the newest readable trusted runtime file from `/asr/` or `/allscan/` for
  bridge activity, connected clients, Zello talkers, and D-Star client data.
- Existing companion daemons can continue writing to their established
  `/allscan/` paths after ASR moves to `/asr/`.
- Uses a strict runtime-filename allowlist and reports the file actually used
  in diagnostics.

## Architecture and recovery

- Keeps stock `/allscan/` available while installing and maintaining ASR under
  `/asr/`.
- Migrates an existing in-place ASR overlay safely and removes obsolete
  web-accessible `/allscan-old/` trees while retaining rollback data in the
  protected backup.
- Preserves users, password hashes, permissions, Favorites, database, node
  settings, ASR settings, bridge settings, friendly names, map cache, uploaded
  logos, and protected secrets.
- Shares the preserved AllScan data between the two interfaces while keeping
  their browser login sessions separate.
- Offers an explicit installer choice to require login for every stock
  `/allscan/` visit, including read-only monitoring.
- Adds a validated rollback control to Reimagined Settings for up to the five
  newest previous ASR versions. Rollback has its own confirmation button and
  does not use Save.

## Dashboard and bridge operation

- Retains authoritative SawStat timers and the accepted direct/adjacent/total
  topology-count formula.
- Retains public four-digit AllStar node selection beginning at `2000` while
  keeping private nodes `1000` through `1999` out of public lookup behavior.
- Adds a separate **DMR Net Bridge** card for a dedicated, tunable DMR bridge.
  Its Connect action selects the entered talkgroup and links the configured
  private bridge node; Disconnect removes both the DMR path and the AllStar
  link.
- Supports per-bridge DMR control paths, ABInfo files, node aliases, and
  dedicated bridge instances rather than hard-coding one DMR installation.
- Shows Idle, TX Active, and Relay from real bridge evidence, with `-` when no
  talker identity applies.
- Keeps the fixed DMR Home Bridge separate from the tunable DMR Net Bridge.
- Preserves unlimited bridge-client lists, recent-transmit sorting, stale
  client removal, and secret-safe bridge diagnostics.

## Administration and mobile use

- Makes the Configs and Users pages usable on phones in portrait and landscape.
- Renames the Configs page heading to **Manage Configs**.
- Adds a mobile-friendly **Help & Instructions** page covering dashboard
  controls, Favorites, access, bridge cards, bridge setup, DMR Net Bridge
  operation, Lookup and map behavior, update notices, rollback, diagnostics,
  and hard refreshes.
- Moves **Maintain Friendly Names** into the Bridge Cards section and removes
  the separate expandable Friendly Names panel.
- Fixes uploaded and legacy header-logo paths across Settings, Help, Lookup,
  Performance Stats, Configs, and Users.
- Preserves uploaded logos during migration from an in-place overlay.
- Clears a transient Favorites loading error after a later successful retry.

## Updates and safety

- Adds a low-frequency, server-side release checker and a prominent cached
  update notice on the main dashboard.
- Shows the installed and available versions, release-notes link, package name,
  and SHA-256 checksum.
- Never installs an update automatically; installation still requires manual
  approval and the normal checksum, backup, validation, and rollback workflow.
- Keeps the previous valid cached result when GitHub is offline, rate-limited,
  or returns an incomplete release.
- Does not expose QRZ, TGIF, bridge, AllScan, AMI, or other credential values.

## Release gate

Publication requires successful package lint and self-tests, side-by-side
migration and rollback testing, KE7WIL regression first, final Thomas and Del
Webb regression, a fresh-install baseline, and Josh's explicit approval.
