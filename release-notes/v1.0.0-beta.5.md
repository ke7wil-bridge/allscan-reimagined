# AllScan Reimagined v1.0.0 Beta 5

Beta 5 focuses on installer polish, admin configurability, mobile admin layout fixes, and optional DMR/TGIF connected-client setup.

## Highlights

- Added a real Admin > Reimagined Settings page.
- Reimagined Settings can update the header title, browser title behavior, logo, bridge visibility, bridge node numbers, and friendly bridge names.
- Improved mobile portrait containment for the Cfgs and Users admin pages while preserving the Reimagined admin header.
- Installer now asks for friendly bridge card names and detail labels when bridge services are detected.
- Installer offers optional DMR/TGIF connected-client tracking when a DMR bridge is enabled.
- TGIF credentials are stored only in `/root/tgif-login.env` with root-only permissions.
- TGIF token refresh installs `tgif-refresh-token.service` and `tgif-refresh-token.timer` with `OnBootSec=2min` and `OnUnitActiveSec=4h`.

## Notes

- Existing AllScan users, passwords, permissions, Favorites, node settings, and `/etc/allscan/allscan.db` are preserved.
- Existing `/etc/allscan-reimagined/config.json` is preserved unless reconfiguration is forced.
- D-Star status remains supported. D-Star connected-client detail is still deferred until real external D-Star talker data is verified.
- Zello talkers only appear after someone talks from Zello into the channel.
